AI ասիստենտները դառնում են առաջին այցելուն, որին հանդիպում է պրոդուկտը։ Երբ հիմնադիրը
խնդրում է ChatGPT-ին կամ Claude-ին գտնել ստուդիա, որը կուղղի vibe-coded հավելվածը, ձեր
սայթը քրոլում է, գները կարդում և որոշում է հիշատակել ձեզ թե ոչ, հենց ագենտը։ Մենք մեզ
տվեցինք պարզ հարց. կարո՞ղ է ագենտն իրականում բիզնես անել մեզ հետ։

Cloudflare-ի [agent-readiness սկաները](https://isitagentready.com) պատասխանեց ուղիղ.
29 100-ից, կարգավիճակը "Bot-Aware"։ Քրոլվել կարելի է, գործել՝ ոչ մի տեղ։ Երկու օր անց
նույն սկաները ցույց է տալիս ուղիղ 100 և բարձրագույն մակարդակը, Level 5 "Agent-Native", իսկ
բալի հետևում կանգնած է աշխատող ագենտային վաճառքի կապուղի։ Պատմում ենք հերթականությամբ,
ներառյալ այն, ինչ հրաժարվեցինք իմիտացնել։

## Կանոնը, որ մեզ ազնիվ պահեց

Ագենտային վեբի ցանկացած ստանդարտ կարելի է խաբել ստատիկ JSON ֆայլով։ Կարելի է
հրապարակել OAuth մետադատա գոյություն չունեցող API-ների համար կամ MCP սերվերի քարտ,
որի հետևում սերվեր չկա։ Բալը կբարձրանա, իսկ մետադատային վստահած ամեն ագենտ իր
թոքենները կվառի՝ աութենտիֆիկացվելով դատարկության մեջ։

Ուստի կանոնը մեկն է. ամեն discovery փաստաթուղթ պարտավոր է ցույց տալ մի բան, որը
պատասխանում է։ Պատրաստ չենք պահել էնդփոինթը, ուրեմն չենք հրապարակում ցուցիչը։

## Քայլ առաջին. հիգիենա (29-ից 50)

Բալի կեսը կազմված է բազային բաներից, որոնք ցանկացած սայթ կարող է թողարկել մեկ երեկոյում.

- robots.txt՝ AI քրոլերների բացահայտ թույլտվությամբ (GPTBot, ClaudeBot, PerplexityBot,
  Google-Extended և մյուսները), գումարած [Content Signals](https://contentsignals.org)
  տողը. search=yes, ai-input=yes, ai-train=yes։ Մի որոգայթ, որն արժե ստուգել ձեր
  Cloudflare զոնայում. Managed robots.txt անջատիչը լուռ ավելացնում է Disallow
  training-քրոլերների համար։ Մենք դա նկատեցինք միայն սեփական աուդիտի ժամանակ։
- llms.txt, ընկերության կառուցվածքային ամփոփում։ Արժե թողարկել, բայց իմացեք
  վիճակագրությունը. ինդուստրիայի չափումներով AI քրոլերները այս ֆայլը դեռ գրեթե չեն
  կարդում։ Այն ոչինչ չի արժենում և օգնում է AI մշակման միջավայրերին, այնպես որ
  հրապարակեք, պարզապես ռազմավարություն մի կառուցեք դրա վրա։
- Link վերնագրեր (RFC 8288) և sitemap, մեքենայաընթեռնելի ցուցիչներ վերը թվարկվածի վրա։
- Agent Skills՝ /.well-known/agent-skills/ հասցեում. երկու սովորական markdown սքիլ,
  մեկը մեր կանոնական փաստերով, երկրորդը՝ ինչպես ճիշտ գնառաջարկ խնդրել հրահանգով։

Այս անցումը մեզ բարձրացրեց 29-ից 50։ Մնացած բոլոր կարմիր չեկերն ասում էին նույն բանը.
դուք չունեք API, չունեք MCP սերվեր, և ագենտը աութենտիֆիկացվելու տեղ չունի։ Սկաները
ճիշտ էր, և մենք կառուցեցինք պակասը։

## Քայլ երկրորդ. իսկական ագենտային գեյթվեյ (50-ից 79)

Մենք բարձրացրինք api.relux.works, մեկ Cloudflare Worker, որը դարձավ ստուդիայի
ագենտային կողմը.

- POST /v1/inquiries, վալիդացվող հայտերի էնդփոինթ։ Ագենտը օգտատիրոջ անունից ուղարկում
  է բրիֆ, նոտիֆիկացիան վայրկյանների ընթացքում ընկնում է մեր Telegram, մարդը
  պատասխանում է մեկ աշխատանքային օրվա ընթացքում։
- MCP սերվեր /mcp-ում (Streamable HTTP, stateless, մոտ երկու հարյուր տող ձեռքով, առանց
  SDK) երկու թուլով. get_services_pricing և request_project_quote։ MCP աջակցող
  ցանկացած ասիստենտ կարող է միանալ և, օգտատիրոջ համաձայնությամբ, վարձել մեզ։
- OpenAPI 3.1, դոկ-էջ և health էնդփոինթ, որ REST մակերեսը նույնպես գտնվի։
- OAuth 2.0 ագենտների համար. մետադատա RFC 8414-ով, բաց դինամիկ գրանցում (RFC 7591) և
  client_credentials գրանտ կարճակյաց ES256 JWT-ներով։ Հաճելի դետալ. ամբողջը stateless
  է։ Կլիենտի գաղտնիքը հաշվարկվում է որպես HMAC client_id-ից, ուստի թոքեն-էնդփոինթը
  ստուգում է creden-ները առանց տվյալների բազայի։ Աութենտիֆիկացիան ընտրովի է by design.
  անանուններին խիստ rate-լիմիտներ, գրանցված ագենտներին՝ առատ, իսկ ամբողջ ֆլոուն
  նկարագրված է /auth.md-ում։

Առաջին ծայրից ծայր թեստը արդարացրեց ամբողջ ձեռնարկումը։ Ասիստենտը MCP-ով կանչեց
request_project_quote, գեյթվեյը ստուգեց բրիֆը, և նոտիֆիկացիան Telegram հասավ մոտ երկու
վայրկյանում։ Որակավորված լիդը մեզ հասավ առանց մեկ մարդու մեր կողմից։

## Քայլ երրորդ. դիստրիբուցիա

Էնդփոինթը, որը ոչ ոք չի գտնում, միևնույն է, որ գոյություն չունի։ Ուստի հրապարակվեցինք
բոլոր կապուղիներում, որտեղ մեր ներկայությունը ազնիվ է.

| Կապուղի | Ինչ կա այնտեղ |
|---|---|
| Պաշտոնական MCP Registry | works.relux/agent-gateway, DNS վերիֆիկացիա |
| ClawHub (OpenClaw) | clawhub install relux-works |
| Hermes tap | hermes skills tap add relux-works/agent-skills |
| HermesHub (ARD) | /.well-known/ai-catalog.json, ինդեքսավորվում է ավտոմատ |
| Well-known-ներ apex-ում | API կատալոգ (RFC 9727), MCP սերվերի քարտ, սքիլների ինդեքս, OAuth մետադատա |
| Հենց էջերը | WebMCP թուլեր navigator.modelContext-ով բրաուզերային ագենտների համար |

Հրապարակեցինք նաև DNS-AID գրառումներ (_index._agents և _mcp._agents), որոնք ցույց են
տալիս գեյթվեյը։ Ստանդարտը դեռ վաղ դրաֆթ է, բայց գրառումներն իրական են, ռեզոլվ են
լինում գլոբալ և ավելացվում են մեկ րոպեում։

## Վերջին մղոնը. 79-ից 100

Վերջին բալերը մաքուր բծախնդրություն էին։ Փակեցինք DNSSEC վստահության շղթան (DS
գրառումը ռեգիստրատորի մոտ, այժմ DNS-AID գրառումները վալիդացվում են AD=true-ով) և
ամբողջացրինք agent_auth մետադատան, որ ագենտը տեսնի անանուն գրանցման ամբողջական
մեթոդը՝ ներառյալ claim URI-ն։ Մեքենաները կարդում են մանր տառերը։

## Ինչ ենք գիտակցաբար թողնում կարմիր

- Կոմերս-պրոտոկոլները (x402, ACP, UCP)։ Մենք զամբյուղով չենք վաճառում, և ձևացնելը
  կխաբեր շոփինգ-ագենտներին։
- Մնացած ամեն ինչը, ինչը պահանջում է էնդփոինթներ, որոնք մենք չենք պահում։

Չեկը, որը կանաչում է միայն մեքենաներին ստելով, թող մնա կարմիր։ Ագենտները լավ
հիշողություն ունեն։

## Եզրակացություններ հիմնադրի համար

1. Ագենտային տրաֆիկը իրական է և կոնվերտ է լինում յուրովի։ Այս այցելուն կարդում է
   llms.txt և անտեսում ձեր hero-անիմացիան։
2. Հիգիենայի մեկ երեկոն (ռոբոտների կանոններ, content signals, sitemap, llms.txt, Link
   վերնագրեր) մուտքի տոմսն է։ Թողարկեք այս շաբաթ։
3. Կուտակվող քայլը ագենտային գործողությունն է. մեկ էնդփոինթ, որը թույլ է տալիս
   ասիստենտին անել այն, ինչի համար գոյություն ունի ձեր բիզնեսը։
4. Discovery փաստաթղթեր հրապարակեք միայն այն բանի համար, ինչը պատասխանում է։
   Կարգո-կուլտ մետադատան ագենտային վեբի բացասական SEO-ն է։
5. Ստուգեք ձեր CDN-ի լռելյայն բոտ-քաղաքականությունը։ Այն կարող է լուռ արգելափակել
   հենց այն քրոլերները, որոնք ձեզ ամենաշատն են պետք։

և [vibe-coded հավելվածների փրկությունից](/hy/vibe-code-rescue/) մինչև
[փաբլիշինգի ամբողջական վարում](/hy/app-publishing/)։ Իսկ եթե սեղմելը ձանձրալի է,
ուղղորդեք ձեր ասիստենտին https://api.relux.works/mcp, թող խոսի մերի հետ։
