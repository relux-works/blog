AI ასისტენტები ხდებიან პირველი ვიზიტორი, რომელსაც პროდუქტი ხვდება. როცა დამფუძნებელი
ChatGPT-ს ან Claude-ს სთხოვს, იპოვოს სტუდია, რომელიც vibe-coded აპლიკაციებს ასწორებს,
თქვენს საიტს კრაულავს, ფასებს კითხულობს და წყვეტს, გახსენოთ თუ არა, სწორედ აგენტი.
ჩვენ საკუთარ თავს მარტივი კითხვა დავუსვით: შეუძლია თუ არა აგენტს რეალურად გააკეთოს
ჩვენთან ბიზნესი?

Cloudflare-ის [agent-readiness სკანერმა](https://isitagentready.com) პირდაპირ
გვიპასუხა: 29 100-დან, სტატუსი "Bot-Aware". კრაულვა შესაძლებელია, მოქმედება კი
არსად. ორი დღის შემდეგ იგივე სკანერი აჩვენებს ზუსტად 100-ს და უმაღლეს დონეს, Level 5
"Agent-Native", ქულის უკან კი მომუშავე აგენტური გაყიდვების არხი დგას. მოვყვებით
თანმიმდევრობით, იმის ჩათვლით, რისი იმიტაციაც არ ვისურვეთ.

## ხაზი, რომელიც თავიდანვე გავავლეთ

აგენტური ვების ნებისმიერი სტანდარტი შეიძლება მოტყუვდეს სტატიკური JSON ფაილით.
შეიძლება გამოაქვეყნო OAuth მეტადატა არარსებული API-ებისთვის ან MCP სერვერის ბარათი,
რომლის უკან სერვერი არ დგას. ქულა გაიზრდება, ხოლო ყოველი აგენტი, რომელიც მეტადატას
ენდობა, საკუთარ ტოკენებს დაწვავს სიცარიელეში ავთენტიფიკაციისას.

ამიტომ წესი ერთია: ყოველი discovery დოკუმენტი ვალდებულია მიუთითოს იმაზე, რაც
პასუხობს. თუ ენდპოინტის შენახვისთვის მზად არ ვართ, მაშინ მაჩვენებელს არ ვაქვეყნებთ.

## ნაბიჯი პირველი: ჰიგიენა (29-დან 50-მდე)

ქულის ნახევარი შედგება საბაზისო რაღაცებისგან, რომლებსაც ნებისმიერი საიტი ერთ საღამოში
გამოუშვებს:

- robots.txt AI კრაულერების აშკარა ნებართვით (GPTBot, ClaudeBot, PerplexityBot,
  Google-Extended და დანარჩენები), პლუს [Content Signals](https://contentsignals.org)
  ხაზი: search=yes, ai-input=yes, ai-train=yes. ერთი ხაფანგი, რომელიც ღირს შეამოწმოთ
  თქვენს Cloudflare ზონაში: Managed robots.txt გადამრთველი ჩუმად ამატებს Disallow-ს
  training-კრაულერებისთვის. ჩვენ ეს მხოლოდ საკუთარი აუდიტისას შევნიშნეთ.
- llms.txt, კომპანიის სტრუქტურირებული შეჯამება. გამოშვება ღირს, მაგრამ იცოდეთ
  სტატისტიკა: ინდუსტრიის გაზომვებით AI კრაულერები ამ ფაილს ჯერ თითქმის არ კითხულობენ.
  ის არაფერი ღირს და AI განვითარების გარემოებს ეხმარება, ასე რომ გამოაქვეყნეთ,
  უბრალოდ სტრატეგიას ნუ ააშენებთ მასზე.
- Link ჰედერები (RFC 8288) და sitemap, მანქანურად წაკითხვადი მაჩვენებლები ყველაფერზე
  ზემოთ ჩამოთვლილზე.
- Agent Skills მისამართზე /.well-known/agent-skills/: ორი ჩვეულებრივი markdown სქილი,
  ერთი ჩვენი კანონიკური ფაქტებით, მეორე ინსტრუქციით, როგორ მოითხოვო შეთავაზება
  სწორად.

ამ გავლამ 29-დან 50-მდე აგვიყვანა. ყველა დარჩენილი წითელი ჩეკი ერთსა და იმავეს
ამბობდა: არ გაქვთ API, არ გაქვთ MCP სერვერი და აგენტს ავთენტიფიკაციის ადგილი არ აქვს.
სკანერი მართალი იყო, და ჩვენ ავაშენეთ ის, რაც აკლდა.

## ნაბიჯი მეორე: ნამდვილი აგენტური გეითვეი (50-დან 79-მდე)

ავაგეთ api.relux.works, ერთი Cloudflare Worker, რომელიც სტუდიის აგენტური მხარე გახდა:

- POST /v1/inquiries, ვალიდირებადი განაცხადების ენდპოინტი. აგენტი მომხმარებლის
  სახელით აგზავნის ბრიფს, შეტყობინება წამებში ვარდება ჩვენს Telegram-ში, ადამიანი კი
  პასუხობს ერთ სამუშაო დღეში.
- MCP სერვერი /mcp-ზე (Streamable HTTP, stateless, დაახლოებით ორასი ხაზი ხელით, SDK-ს
  გარეშე) ორი ტულით: get_services_pricing და request_project_quote. MCP-ის მხარდამჭერ
  ნებისმიერ ასისტენტს შეუძლია დაუკავშირდეს და, მომხმარებლის თანხმობით, დაგვიქირაოს.
- OpenAPI 3.1, დოკუმენტაციის გვერდი და health ენდპოინტი, რომ REST ზედაპირიც
  აღმოჩენადი იყოს.
- OAuth 2.0 აგენტებისთვის: მეტადატა RFC 8414-ით, ღია დინამიკური რეგისტრაცია
  (RFC 7591) და client_credentials გრანტი ხანმოკლე ES256 JWT-ებით. სასიამოვნო
  დეტალი: ეს ყველაფერი stateless-ია. კლიენტის საიდუმლო ითვლება როგორც HMAC
  client_id-სგან, ამიტომ ტოკენ-ენდპოინტი მონაცემთა ბაზის გარეშე ამოწმებს
  კრედენშალებს. ავთენტიფიკაცია არჩევითია by design: ანონიმებს მკაცრი rate-ლიმიტები
  ხვდებათ, რეგისტრირებულ აგენტებს კი ხელუხლებელი, მთელი ფლოუ კი /auth.md-შია
  აღწერილი.

პირველმა ბოლომდე ტესტმა მთელი წამოწყება გაამართლა. ასისტენტმა MCP-ით გამოიძახა
request_project_quote, გეითვეიმ ბრიფი შეამოწმა, და შეტყობინება Telegram-ში დაახლოებით
ორ წამში მოვიდა. კვალიფიციური ლიდი ჩვენამდე ისე მოვიდა, რომ ჩვენი მხრიდან არც ერთი
ადამიანი არ ჩართულა.

## ნაბიჯი მესამე: დისტრიბუცია

ენდპოინტი, რომელსაც ვერავინ პოულობს, თითქოს არც არსებობს. ამიტომ გამოვქვეყნდით ყველა
არხში, სადაც ჩვენი ყოფნა პატიოსანია:

| არხი | რა არის იქ |
|---|---|
| ოფიციალური MCP Registry | works.relux/agent-gateway, DNS ვერიფიკაცია |
| ClawHub (OpenClaw) | clawhub install relux-works |
| Hermes tap | hermes skills tap add relux-works/agent-skills |
| HermesHub (ARD) | /.well-known/ai-catalog.json, ინდექსირდება ავტომატურად |
| Well-known-ები apex-ზე | API კატალოგი (RFC 9727), MCP სერვერის ბარათი, სქილების ინდექსი, OAuth მეტადატა |
| თავად გვერდები | WebMCP ტულები navigator.modelContext-ით ბრაუზერული აგენტებისთვის |

ასევე გამოვაქვეყნეთ DNS-AID ჩანაწერები (_index._agents და _mcp._agents), რომლებიც
გეითვეისკენ მიუთითებენ. სტანდარტი ჯერ ადრეულ დრაფტშია, სამაგიეროდ ჩანაწერები
ნამდვილია, გლობალურად რეზოლვდება და ერთ წუთში ემატება.

## ბოლო მილი: 79-დან 100-მდე

ბოლო ქულები სუფთა გულმოდგინება აღმოჩნდა. დავხურეთ DNSSEC ნდობის ჯაჭვი (DS ჩანაწერი
რეგისტრატორთან, ახლა DNS-AID ჩანაწერები AD=true-თი ვალიდირდება) და დავასრულეთ
agent_auth მეტადატა, რომ აგენტმა სრული ანონიმური რეგისტრაციის მეთოდი დაინახოს,
claim URI-ს ჩათვლით. მანქანები წვრილ შრიფტსაც კითხულობენ.

## რამდენად ღირს სინამდვილეში იდეალური ქულა

უხერხული ნაწილის დროა. სკანერი აშენებულია კომპანიის მიერ, რომელიც ყიდის
გამოსწორებებს: Markdown for Agents ფასიან გეგმაში ცხოვრობს, Pay Per Crawl მათი
მარკეტპლეისია, ხოლო Managed robots.txt გადამრთველი, რომელმაც ჩუმად დაბლოკა training
კრაულერები ჩვენსავე ზონაში, ნაგულისხმევად ჩართული მოდის. Cloudflare არაფერს
არანჟირებს; მათი ნამდვილი ძალა იმაშია, რომ ვების მეხუთედზე წყვეტენ, რომელი
კრაულერები გაივლიან კარში და რა ფასად. მზადყოფნის ქულა ამ ბიზნესისთვის შესანიშნავი
ძაბრია.

და თავად ქულა არაფერს ძრავს. არცერთი ძრავა მისით არ არანჟირებს. ასისტენტების
პასუხები ორ ეტაპად იწყობა: ჯერ კლასიკური ინდექსი (Bing ChatGPT-სთვის, Brave
Claude-სთვის) არჩევს კანდიდატ გვერდებს, შემდეგ აგენტი მათ იწერს და კითხულობს.
ყველაფერი, რაც გამოვუშვით, მეორე ეტაპზე ცხოვრობს. ჩვენი წაკითხვა იაფია და ჩვენთან
მოქმედება შესაძლებელი, პირველ ეტაპს კი ეს არაფერს აძლევს.

ეს საკუთარ თავზე გავზომეთ. ჩვენი მთავარი გვერდი უკვე იმ ინდექსშია, რომლითაც Claude
ეძებს; სერვისული გვერდები იქ ჯერ არ არის, ხოლო ფრაზაზე vibe code rescue იგივე
ინდექსი ნახევარ დუჟინ კონკურენტს აბრუნებს. ამასობაში ყველაზე მნიშვნელოვანი
კრაულერები უკვე აქ არიან: Anthropic-ისა და OpenAI-ის საკუთარი ბოტები ამ საიტს
ყოველდღიურად კითხულობენ მას შემდეგ, რაც კარი გავუღეთ. ლაბორატორიები საკუთარ
ინდექსებს აშენებენ, და იქ ადრე მოხვედრა შეიძლება ნებისმიერ ქულაზე მეტად აღმოჩნდეს
მნიშვნელოვანი.

მაშ რატომ გავაკეთეთ ჩეკლისტი საერთოდ? პირველი დღის წესის გამო. ყოველი
გამოქვეყნებული მაჩვენებელი მიდის იმისკენ, რაც პასუხობს: MCP სერვერი ნამდვილ
განაცხადებს იღებს, OAuth ნამდვილ ტოკენებს გასცემს, გეითვეი თავს იმართლებს როგორც
გაყიდვების არხი, რასაც არ უნდა ფიქრობდეს ნებისმიერი სკანერი. აკეთეთ ჩეკები,
რომლების შემდეგ მუშა ინფრასტრუქტურა რჩება, გამოტოვეთ ისინი, რომლების შემდეგ JSON
ფაილი რჩება, დაზოგილი დრო კი დახარჯეთ იქ, საიდანაც პასუხები ნამდვილად მოდის:
მოსაწყენ ინდექსებში, საზოგადოებებში და გვერდებზე, რომლებიც ციტირებად ღირს.

## რას ვტოვებთ წითლად შეგნებულად

   კითხულობს და თქვენს hero-ანიმაციას უგულებელყოფს.

   ჰედერები) შესვლის ბილეთია. გამოუშვით ამ კვირაში.
3. მზარდი სვლა აგენტური მოქმედებაა: ერთი ენდპოინტი, რომელიც ასისტენტს აძლევს
   საშუალებას გააკეთოს ის, რისთვისაც თქვენი ბიზნესი არსებობს.

   მეტადატა აგენტური ვების უარყოფითი SEO-ა.
